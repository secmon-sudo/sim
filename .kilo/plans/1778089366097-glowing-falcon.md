## Kritik Sorunlar (Pipeline Çalışmasını Engelleyebilir / Veri Kaybı Riski)

### 1. `requirements.txt` Eksik
- **Dosya:** Proje kökü
- **Sorun:** GitHub Actions workflow (`osint-pipeline.yml`) `pip install -r requirements.txt` çalıştırıyor ama dosya yok.
- **Etki:** Pipeline hiç çalışamaz. Streamlit de deploy edilemez.
- **Eksik paketler:** `httpx`, `tenacity`, `psycopg[binary]`, `streamlit`, `pydeck`, `streamlit-agraph`, `tldextract`, `python-dotenv`, `pandas`, `networkx` (muhtemelen), `pytest`, `pytest-mock`

### 2. HeartbeatWorker — Thread Safety İhlali (Kritik)
- **Dosya:** `src/core/heartbeat.py`
- **Sorun:** `_run()` metodu aynı `db_conn` üzerinde `execute()` + `commit()` çalıştırıyor. `psycopg` connection'ları thread-safe değildir.
- **Etki:** Ana thread (Pass C) ile heartbeat thread aynı connection'ı kullanırsa race condition, commit conflict veya connection crash olabilir. Lock kaybı veya DB corrupt veri yazabilir.
- **Blueprint referansı:** §4 — "heartbeat update thread" ayrı connection kullanmalı.

### 3. Supabase Client — Singleton Connection Thread Safety
- **Dosya:** `src/services/supabase_client.py`
- **Sorun:** `_connection` global singleton. Çoklu worker/thread (örneğin heartbeat + pipeline) aynı connection'ı paylaşıyor.
- **Etki:** `psycopg` aynı connection üzerinde eşzamanlı sorgu desteklemez. `InternalError: cannot perform operation: another operation is in progress` hatası alınabilir.

### 4. Pass B — URL Hash Deduplication Eksik
- **Dosya:** `src/pipeline/pass_b_dedup.py`
- **Sorun:** `mark_as_deduped()` TÜM `status = 'raw'` event'leri koşulsuz `deduped` yapıyor. `source_url_hash` kontrolü yok.
- **Etki:** Aynı URL'den gelen duplicate event'ler LLM classification'a kadar ilerler. Blueprint'te "In a full implementation..." notu var ama bu kritik bir eksiklik.

### 5. Storyline Linking — `None` Datetime Hatası
- **Dosya:** `src/core/storyline.py`
- **Sorun:** `should_link_storyline()` `event_a["occurred_at_est"] - event_b["occurred_at_est"]` yapıyor. Eğer biri `None` ise `TypeError`.
- **Etki:** Pass D crash edebilir, event scoring yapılamaz.

## Yüksek Öncelikli Sorunlar (Veri Bütünlüğü / Güvenlik)

### 6. HTML Injection / XSS Riski (Streamlit)
- **Dosya:** `streamlit_app/components/alert_feed.py`
- **Sorun:** `source_title`, `event_type`, `anchor_name_norm` doğrudan f-string HTML'e interpolasyon ediliyor. `unsafe_allow_html=True` var.
- **Etki:** LLM tarafından classify edilmiş zararlı içerik (`<script>`, `onclick=` vb.) Streamlit dashboard'da çalıştırılabilir.
- **Aynı risk:** `telegram_notifier.py` HTML mesaj formatlamasında da var. Telegram HTML parse_mode'da `<`, `>`, `&` escape edilmeli.

### 7. Telegram Archive Chat Fallback Riski
- **Dosya:** `src/services/telegram_notifier.py`
- **Sorun:** `TELEGRAM_ALERTS_CHAT_ID` yoksa `TELEGRAM_ARCHIVE_CHAT_ID`'ye düşüyor.
- **Etki:** Alert'ler archive kanalına gidebilir. Bu veri gizliliği / operasyonel karışıklık riski.

### 8. LLM Response Çift Encoding
- **Dosya:** `src/pipeline/pass_c_classify.py`, `src/core/llm_client.py`
- **Sorun:** `llm_raw_output = json.dumps(result.get("response", {}))` — `result["response"]` zaten dict. Bu dict string'e çevrilip JSONB column'a yazılıyor. Okunurken `json.loads` tekrar gerekir.
- **Etki:** `events.llm_raw_output` JSONB içinde string-encoded JSON saklanır. Query karmaşıklığı artar.

### 9. Pass F Archive — `occurred_at_est` NULL Event'ler Asla Archive Edilemez
- **Dosya:** `src/pipeline/pass_f_archive.py`
- **Sorun:** `WHERE occurred_at_est < NOW() - INTERVAL '%s days'` — `NULL` ile karşılaştırma her zaman `UNKNOWN` (false) döner.
- **Etki:** `occurred_at_est` olmayan event'ler (LLM zaman çıkarımı yapamadıysa) DB'de sonsuza kadar kalır.

### 10. Auto-Refresh Meta Tag — State Kaybı
- **Dosya:** `streamlit_app/app.py`
- **Sorun:** `<meta http-equiv="refresh">` kullanılıyor. Bu tarayıcı seviyesinde refresh'tir; Streamlit widget state (toggle, slider, search) tamamen sıfırlanır.
- **Etki:** Kullanıcı filtreleri her 60 saniyede sıfırlanır.

## Orta Öncelikli Sorunlar (Mantık Hataları / Verimsizlik)

### 11. Pass D N+1 Query Problemi
- **Dosya:** `src/pipeline/pass_d_score.py`
- **Sorun:** `score_single_event()` her event için ayrı `recent_events` sorgusu çalıştırıyor. 100 event = 100 ek sorgu.
- **Etki:** Pipeline yavaşlar. Her 30 dakikada 100 event × 100 query = 10.000 query.

### 12. `compute_url_hash` — Query String Atılıyor
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** `normalized = url.strip().lower().split("?")[0].split("#")[0]` — query string tamamen atılıyor.
- **Etki:** `https://example.com/news?id=123` ve `https://example.com/news?id=456` aynı hash'e sahip olur. Farklı makaleler duplicate sayılır.

### 13. `is_noise` Substring Match — False Positive
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** `if pattern.lower() in text_lower:` — "drill" kelimesi "drilling" içinde match eder.
- **Etki:** Gerçek haberler (örneğin "drilling rights dispute") yanlışlıkla noise olarak atılır.

### 14. `normalize_anchor` — IATA Match Büyük/Küçük Harf Hassas
- **Dosya:** `src/core/anchor.py`
- **Sorun:** `re.match(r"^[A-Z]{3,4}$", raw_text)` — LLM çıktısı `"cai"` (küçük harf) ise match etmez.
- **Etki:** Doğrudan IATA/ICAO eşleşmesi kaçırılabilir. Sonra trigram fuzzy match'e düşer (daha düşük confidence).

### 15. Telemetry Dashboard — `llm_router` Hiç Geçilmiyor
- **Dosya:** `streamlit_app/app.py`, `streamlit_app/components/telemetry_dashboard.py`
- **Sorun:** `app.py` `render_telemetry(stats)` çağrıyor ama `llm_router` parametresi vermiyor. Dashboard her zaman hardcoded 1000 tabanlı quota gösterir.
- **Etki:** Gerçek LLM kota durumu dashboard'da görünmez.

### 16. `settings.json` Sabit Kodlanmış Değerler
- **Dosya:** `src/pipeline/pass_b_dedup.py`
- **Sorun:** `STALE_LOCK_THRESHOLD_MINUTES = 15`, `MATURATION_WINDOW_HOURS = 2` Python sabiti. `config/settings.json` içinde değerler var (`stale_lock_threshold_minutes`, `maturation_window_hours`) ama kullanılmıyor.
- **Etki:** Ayar değiştirilmek istendiğinde kod değişmeli.

### 17. `canonicalize_text` HTML Regex Riskli
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** `re.sub(r"<[^>]+>", " ", raw_text)` — matematiksel ifadeler (`3 < 5 > 2`) veya e-posta adresleri (`user<at>domain`) yanlış strip edilir.
- **Etki:** Makale içeriği bozulabilir.

### 18. Diversity Score Sabit 0.5
- **Dosya:** `src/pipeline/pass_d_score.py`
- **Sorun:** `compute_confidence()` `diversity_score=0.5` sabit. Gerçek diversity (kaynak çeşitliliği, domain sayısı vb.) hesaplanmıyor.
- **Etki:** Confidence score yapay ve öngörülebilir olur.

### 19. Pass E Reconciliation — "Top-10 Arrays" Eksik
- **Dosya:** `src/pipeline/pass_e_reconcile.py`
- **Sorun:** Blueprint §PASS E'de "clears Top-10 arrays on anchor upgrade" deniyor ama kodda hiçbir "Top-10" yapısı yok.
- **Etki:** Blueprint'te var ama kodda implemente edilmemiş özellik.

### 20. `pass_a` RSS Description Yetersiz
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** Google News RSS sadece title + description getirir. Makale tam içeriği yok.
- **Etki:** LLM classification sadece snippet üzerinden yapılır, doğruluk düşebilir.

## Kullanıcı Geri Bildirimine Dayalı Ek Bulgular

### 29. RSS pubDate Filtreleme Yok — Eski Haberler Birikiyor
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** `fetch_rss_feed()` RSS `<pubDate>` değerini parse etmiyor, hiçbir tarih filtresi uygulamıyor. Google News ve Reddit RSS'lerinde 2-3 ay önceki haberler de dönebilir.
- **Etki:** DB'de `raw` event birikimi patlar. Her 30 dakikada aynı eski haberler tekrar çekilir, Pass B'den geçer, gereksiz LLM call'ları tüketir.
- **Kullanıcı talebi:** Maksimum 3-4 günlük zaman penceresi ile tarama yapılmalı.
- **Öneri:** `fetch_rss_feed`'e `pub_date` parse (`email.utils.parsedate_to_datetime`) ve cutoff filter (`now_utc - max_age_days`) ekle. `settings.json`'da `max_article_age_days` konfigürasyonu tanımla.

### 30. İçerik Tabanlı Deduplication Yok — Aynı Haber Farklı Kaynaklardan Giriyor
- **Dosya:** `src/pipeline/pass_a_ingest.py`, `src/pipeline/pass_b_dedup.py`
- **Sorun:** Şu anki dedup sadece `source_url_hash` üzerinden çalışıyor. Aynı haber farklı domain'lerden (`bbc.com/news/123`, `cnn.com/world/abc`) farklı URL hash'lerle gelirse duplicate sayılmaz.
- **Etki:** Aynı olay 5-10 farklı kaynaktan ayrı event olarak DB'ye girer, LLM classification maliyeti katlanır.
- **Kullanıcı talebi:** Aynı haber farklı kaynaklardan geldiğinde duplikasyon önlenmeli.
- **Öneri:** Pass A veya Pass B'ye content-based dedup ekle. Seçenekler:
  - **A:** `canonical_text` üzerinde MinHash/LSH benzerlik skoru (en robust ama maliyetli).
  - **B:** Normalize edilmiş `source_title` + `event_type` (henüz belli değil) + `occurred_at_est` gün bazında gruplama.
  - **C:** Pragmatik — `canonical_text`'in ilk 200 karakteri üzerinde `SORENCE`/`simhash` ile yaklaşık eşleşme (en uygun).
- **Not:** LLM classification henüz yapılmamışken content-based dedup yapmak gerekir. Pass A sonrası, Pass B öncesinde title+description similarity kontrolü yapılabilir.

### 31. USA Ağırlıklı Haber Dağılımı — Coğrafi Dengesizlik
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** Google News RSS sorgusu `hl=en&gl=US&ceid=US:en` ile ABD merkezli çalışıyor. Emergency keywords listesi ABD vakalarını daha sık yakalar.
- **Etki:** Avrupa, Ortadoğu, Afrika, Asya'daki olaylar under-represented. Dashboard dengesiz görünür.
- **Kullanıcı talebi:** USA ağırlıklı çok fazla haber geliyor.
- **Öneri:**
  - RSS fetch'e coğrafi ağırlık parametresi ekle (örn. `gl=US` yerine `gl=INT` veya çoklu `gl` sorguları).
  - Ya da keywords config'e bölgesel query setleri ekle (Ortadoğu, Avrupa, Afrika, Asya-Pasifik).
  - Ya da Pass A sonrası `source_domain` country detection ile coğrafi filtre uygula — ama bu under-representation'ı çözmez, sadece USA'yi azaltır (ilginç olayları kaçırma riski).
  - **En iyi yaklaşım:** Her ülke/bölge için ayrı RSS sorguları ve domain-based geographic tagging ile dengeli ingestion.

### 32. Makale Tam Metni Eksik — LLM Classification Doğruluğu Düşük
- **Dosya:** `src/pipeline/pass_a_ingest.py`
- **Sorun:** Google News RSS sadece `title` + `description` (snippet, ~1-2 cümle) sağlar. Makalenin tam içeriği (`full_text`) çekilmiyor.
- **Etki:** LLM classification sınırlı bağlamla çalışır. Özellikle karmaşık olaylarda (`geopolitical_conflict`, `terrorism` vs.) yanlış classify riski artar.
- **Kullanıcı talebi:** LLM free sources kullanıldığı için tüm haber metnini alamıyor.
- **Öneri:**
  - **Seçenek A (ücretsiz):** `trafilatura`, `newspaper3k` veya `readability-lxml` ile makale URL'sinden tam metin çekmek. Ek bağımlılık ve zaman maliyeti var (~1-3 sn/article). Ancak free tier LLM kullanımında classification doğruluğu kritik.
  - **Seçenek B (mevcut + iyileştirme):** RSS description'ı daha iyi kullanmak + LLM prompt'ta "bu bir snippet" olduğunu belirtmek.
  - **Seçenek C (hybrid):** Sadece title+description ile ön-sınıflandırma yap, yüksek öncelikli event'ler için (örn. `alert_tier >= ALERT`) tam metin çek.
  - **Kullanıcı notu:** Free sources kullanıldığı için tam metin alınamıyor. Bu durumda Seçenek B veya C pragmatik.

## Kapsam Genişletme — Yeni Tehdit Kategorileri (2026-05-06)

### 33. Drone Saldırıları (Özellikle Havalimanlarına)
- **Mevcut durum:** `drone incursion`, `drone attack` keyword'leri var ama yetersiz.
- **Eksik keyword'ler:** `drone strike airport`, `UAV attack`, `unmanned aerial vehicle`, `drone bombing`, `drone swarm`, `counter-drone`, `drone sighting runway`.
- **Talep:** Havalimanlarına yapılan drone saldırıları yüksek öncelikli, genel drone haberleri (ticari, hobi) değil.
- **Öneri:**
  - `keywords.json` → `emergency_keywords` bölümüne drone spesifik terimler ekle.
  - LLM prompt'ta `drone_incursion` alt tipi için özel yönerge: "sadece havalimanı, askeri üs, kritik altyapı hedefli drone saldırılarını raporla, hobi drone veya ticari drone kazalarını raporlama".
  - Noise filter: `drone delivery`, `drone photography`, `racing drone` gibi kelimeler ekle.

### 34. Uzun Süren Çatışmalar (İran-Amerika, Ukrayna-Rusya, Sudan, Mali, vs.)
- **Mevcut durum:** `geopolitical_keywords` bölümünde `iran israel`, `ukraine russia`, `sudan conflict` var ama genel çatışma kelimeleri eksik.
- **Eksik keyword'ler:** `war escalation`, `ceasefire broken`, `humanitarian crisis`, `refugee crisis`, `airstrike civilian`, `artillery shelling`, `frontline`, `troop buildup`, `border clash`, `armed conflict`.
- **Talep:** Uzun süren çatışmalardaki kritik gelişmeler (askeri hareket, sivil hedef vurulması, ateşkes ihlali) takip edilmeli.
- **Öneri:**
  - `geopolitical_keywords` genişlet: bölgesel çatışma isimleri + genel askeri terimler.
  - LLM classifier'a yeni `event_type`: `war_escalation`, `ceasefire_violation`, `civilian_casualties`.
  - `event_type_catalog` migration'una yeni tipler ekle.

### 35. Havayolu / Havalimanı Personeline Yönelik Saldırılar
- **Mevcut durum:** `airline crew attack`, `pilot assault`, `flight attendant attack` var ama yetersiz.
- **Eksik keyword'ler:** `ground staff stabbed`, `cabin crew assaulted`, `airport worker killed`, `baggage handler attacked`, `TSA officer assaulted`, `security guard airport`, `check-in agent attacked`, `ramp agent injured`, `air traffic controller threat`.
- **Talep:** Normal bir kişinin yolda bıçaklanması ilgi çekmez ama **havayolu personeli** bıçaklanırsa/sekse önemli. KitleSEL saldırılar genel olarak önemli.
- **Fark:** Rol/sector spesifik olaylar kişi spesifik olaylardan ayrılmalı.
- **Öneri:**
  - `keywords.json`'a personel spesifik terimler ekle.
  - Noise filter: `man stabbed`, `woman attacked`, `street fight` (rol belirtilmemişse noise).
  - LLM prompt güncelle: "aviation sector employee (pilot, cabin crew, ground staff, baggage handler, TSA, security, check-in agent, air traffic controller) attacked → yüksek öncelik. Genel sokak saldırısı → düşük öncelik. KitleSEL saldırı (3+ ölü/yaralı) → her zaman yüksek öncelik".

### 36. KitleSEL Saldırılar / Mass Casualty Events (Sokakta Bile Olsa)
- **Mevcut durum:** `active shooter`, `bomb threat` var ama `mass` kelimesi yok.
- **Eksik keyword'ler:** `mass shooting`, `mass stabbing`, `mass casualty`, `multiple fatalities`, `massacre`, `suicide bombing`, `vehicle ramming`, `IED explosion`, `multiple injured`, `dozens killed`.
- **Talep:** Kişi sayısı eşiği aşıldığında lokasyon önemsiz olabilir. 3+, 5+, 10+ ölü/yaralı olan her saldırı önemli.
- **Öneri:**
  - `keywords.json`'a mass casualty terimleri ekle.
  - LLM classifier'a özel talimat: "ölü/yaralı sayısını çıkar. Eğer 3+ ölü veya 10+ yaralı varsa severity_score +20 bonus".
  - `pass_d_score.py`'e casualty-based bonus logic ekle.

### 37. Afrika Terör Olayları (Mali, Burkina Faso, Nijer, Somali, Nijerya, vs.)
- **Mevcut durum:** `terrorism` genel keyword'ü var ama Afrika spesifik yok.
- **Eksik keyword'ler:** `jihadist attack`, `Boko Haram`, `Al-Shabaab`, `Sahel crisis`, `Mali attack`, `Burkina Faso attack`, `Niger coup`, `Somalia bombing`, `Nigeria abduction`, `ISIS Africa`, `Ansar al-Islam`, `JNIM`, `Wagner Africa`, `African terrorism`.
- **Talep:** Afrika'daki cihatçı/terör saldırıları çok önemli.
- **Öneri:**
  - `keywords.json`'a Afrika terör spesifik terimler ekle (en + fr + ar + swahili).
  - Geo regions'a `gl=NG` (Nijerya), `gl=ZA` (Güney Afrika), `gl=KE` (Kenya) ekle.
  - `event_type_catalog`'a `african_terrorism`, `insurgency_attack`, `extremist_violence` ekle.

### 38. Otel / Resort Saldırıları (Turizm Hedefli)
- **Mevcut durum:** `hotel attack`, `hotel bombing`, `hotel shooting` var.
- **Talep:** Turist otellerine yapılan saldırılar önemli (özellikle Afrika, Ortadoğu, Güneydoğu Asya).
- **Öneri:**
  - `keywords.json`'a `resort bombing`, `beach attack`, `tourist bus attack`, `cruise ship attack` ekle.
  - `anchor_master`'a turist hotspot'ları ekle (Sharm el-Sheikh, Bali, Phuket, Mombasa).
